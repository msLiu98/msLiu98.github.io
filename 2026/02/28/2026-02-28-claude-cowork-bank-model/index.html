Claude Cowork × A股银行财务建模实测 - 从谷歌到招商银行

之前用Claude做了一版Google的财务模型——读10-K、拆收入结构、搭forecast，整个流程跑得很顺（4小时 vs 3天）。做完之后发给首席看，翻了一下，问了一句：这个能做谷歌的，那A股银行它也能做吗？

那A股银行还有谷歌复杂吗？Google那种多业务线、GAAP和Non-GAAP双轨的结构都能处理，于是当天晚上就试了。先拿招商银行开刀，四份年报（2021-2024）扔进去，几轮对话，出来一个8个sheet、422个公式的完整模型。

工作流：简单到有点无聊

整个流程用的是 Claude 桌面端的 Cowork 模式。操作上没有任何技术门槛——不需要写代码，不需要配环境。

第一步

把招行2021-2024四份年报PDF和谷歌的财务模型Excel一起拖进对话窗口。

第二步

一句话prompt：

参考谷歌的财务报告模式，结合招行的年报，我们做个招行的财务模型，历史3年加预测3年

然后它就开始工作了。先翻年报提取数据，对着合并报表一项一项读数，搭建Excel的sheet结构，写公式，做格式。第一版大概几分钟就出来了。

迭代：像跟分析师对需求一样

第一版出来之后打开看了一下，能用，但太粗。资产负债表和利润表都是加总后的简化版，类似你在研报摘要里看到的那种——贷款、存款、净利息收入、手续费、就完了。

于是给了一段反馈：

这个版本太简单了。我需要看到原始的资产负债表和利润表，放在单独的sheet，比如’其他资产’你要把原始的明细带进来。然后用公式从原始表加总到预测表。另外资产质量的数据也要——关注贷款比例、逾期贷款比例、存款成本率这些，分别整理成新的表格。

注意：这段话没有告诉它具体哪些科目、在年报哪一页、数字是多少。

它自己去翻年报的合并报表和管理层讨论章节，把17项资产明细、14项负债明细、五级分类、逾期贷款分层、拨备变动、存款结构（公司活期/定期、零售活期/定期）、NIM分解、各类资产收益率和负债成本率全部提取出来。

最终成果：8张Sheet，422个公式

最终交付的Excel模型包含8个工作表，结构如下：

Sheet 内容
假设 增长驱动、盈利驱动、资本与分红假设，黄底蓝字可调
原始资产负债表 17项资产 + 14项负债 + 权益拆分，完整还原年报原始科目
原始利润表 法定格式，利息收支、手续费收支、投资收益、公允变动等全部明细
资产质量 五级分类、逾期分层、拨备变动、信贷成本率
利差与融资 NIM分解、贷款收益率、存款成本率（按类型）、存款结构
利润表预测 历史年引用原始表（绿色），预测年公式驱动（黑色）
资产负债表预测 贷款/存款/投资独立增长，同业负债残差平衡
关键指标 ROAE、ROAA、NIM、EPS/DPS/BVPS、CET1、存贷比

几个值得一提的细节：

公式联动：预测表的历史列不是硬编码，而是用绿色字体的公式链接回原始报表。比如利润表预测里的FY2024净利息收入，实际上是 ='原始利润表'!E7。改了原始数据，预测表自动更新。

颜色规范：遵循投行标准——蓝色是硬编码输入，黑色是公式，绿色是跨表引用。不用点进去看就知道哪些格子是可以调的假设。

平衡检查：资产负债表底部有Balance Check行，三年历史数据全部为零。这不是巧合——它在搭模型的过程中发现过平衡问题（少数股东权益的处理），自己定位了bug并修复了。

422个公式，0个错误。用LibreOffice自动recalc验证，全部通过。

效率对比：这事以前得做多久？

做过银行模型的人都知道，这活儿最耗时的不是搭结构——结构一个模板就行。耗时的是从年报里抠数。

招行的年报每份两三百页，合并报表的科目有几十项，你要对着PDF里的表格，一个数一个数往Excel里填。填完还要交叉验证——总资产等不等于各项加总？净利润对不对得上？拨备余额和变动表能不能打平？

如果还要做资产质量分析，你得去翻管理层讨论与分析那一章，找五级分类的表、逾期的表、拨备变动的表、存款结构的表、NIM分解的表，分散在不同页面，逐一录入。

保守估计，一个熟练的分析师手工完成这套模型，至少需要一整天——还不算检查和调试公式的时间。

环节 传统手工 Claude Cowork
数据提取 3-4小时 自动完成
模型搭建 2-3小时 自动完成
检查调试 1-2小时 自动recalc验证
人工介入 全程 2-3轮对话反馈
合计 6-8小时 ~30分钟

需要说明的是，这30分钟出来的模型不是”demo级别”的演示品。它的数据精度、公式规范性和结构完整度，基本达到了可以直接拿去做进一步分析的水平。当然，预测假设需要根据你自己的判断调整——这也恰恰是模型该有的样子：假设归人，计算归机器。

几个实操Tips

1. 给参考模板比纯文字描述有效10倍

把谷歌的财务模型Excel作为”参考样板”一起上传了。Claude看到了sheet结构、颜色规范、公式逻辑，直接照着这个标准来做。比你在prompt里写一大段”我要蓝色输入黑色公式绿色引用”有效得多。

2. 先出粗版，再迭代细化

不要试图在第一句prompt里把所有需求说清楚。先让它出一版，打开看看哪里不满意，再用自然语言反馈。它会理解”太简单了需要细化”这种模糊指令，然后自己去年报里找更详细的数据。

3. 中文年报它读得懂

不需要翻译，不需要预处理。直接扔PDF进去就行。它能识别中文科目名称，知道”发放贷款和垫款”对应Loans & Advances，知道”以摊余成本计量的金融资产”是什么意思。

4. 检查很重要，但它自己也会检查

模型里自带Balance Check行，每一年都验证资产=负债+权益。它还会在P&L底部放一个净利润校验行（归母+少数=总净利润）。在构建过程中，它发现过平衡差异并自行修复了——差值恰好等于少数股东权益，是公式引用错了一行。

下一步计划

接下来准备做一个A股银行的系列预测模板——覆盖主要上市银行，参考一致预期的数值做好基础假设，直接生成可用的Excel模型。

模板会发给大家自行二次调整参数，你觉得NIM假设太乐观？改一个数字，整个模型自动重算。觉得信贷成本率该上调？拉一下就行。假设归你，计算归模型。

同时也会把模板放到 GitHub 上，欢迎大家用完之后提反馈、报bug、甚至贡献改进方案，一起把这个东西打磨得更好用。
